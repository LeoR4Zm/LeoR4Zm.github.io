Images

DTC355 Code log images: How images are used in HTML and CSS, with side-by-side examples.

1) <img> vs background-image

HTML image (<img>)

Use when the image is content. It’s in the document flow, accessible by default, can have alt text, and is affected by layout properties like width and max-width.

<img
  src="dog.jpg"
  alt="A dog looking at the camera"
  class="demo-img"
/>

CSS background image

Use when the image is decoration. It’s not read by screen readers (unless you add additional markup), sits behind content, and doesn’t affect element size.

background-image 
.bg-box {
  background-image: url("dog.jpg");
  background-size: cover;
  background-position: center;
  background-repeat: no-repeat;
}
Key differences
  • Semantics & accessibility: <img> conveys content and supports alt. Backgrounds are decorative (no alt)—don’t use them for content that users must perceive.
  • Layout: <img> participates in layout and sizes with its box; background images don’t change element size—they fill the element you give them.
  • Styling: object-fit/object-position style <img>; background-size/background-position/background-repeat style background images.
  • Interaction: <img> can be focused, linked, or lazy-loaded. Backgrounds are harder to make accessible and can’t have alt.

2) Background properties beyond background-image

background-size

Controls how the image fits the box. Common values: cover (fill, may crop), contain (fit inside, no crop), or explicit sizes (e.g., 200px auto).

background-position

Sets the anchor point inside the box (e.g., center, top, left 20%). Useful with cover to choose which part stays visible.

background-repeat

Repeat behavior: repeat, no-repeat, repeat-x, repeat-y. Great for patterns or textures.

More useful ones
  • background-attachment: fixed for a parallax-like effect.
  • Multiple backgrounds: background: url(a), url(b); layered back-to-front.
  • background-clip / background-origin to control painting areas.

3) Dimensions vs resolution (one sentence)

Dimensions are the pixel width × height of an image (e.g., 1200×800), while resolution is pixel density (e.g., 72 or 300 ppi) describing how tightly those pixels are packed when displayed or printed.

4) Formats: similarities & differences (extra credit)

  • GIF — 8-bit palette (256 colors), lossless LZW, supports simple animation and 1-bit transparency; best for tiny animations or very simple graphics; poor for photos.
  • PNG — Lossless, supports full alpha transparency; great for UI, icons, line art, screenshots; larger than JPEG for photos.
  • JPG/JPEG — Lossy compression tuned for photos; small files at acceptable quality; no true transparency; not ideal for sharp text/UI.
  • WebP — Modern format supporting lossy & lossless compression, transparency, and animation; usually smaller than PNG/JPEG with similar quality; widely supported in modern browsers.
  • SVG — Vector graphics (XML); infinitely scalable, small for logos/icons/diagrams; styleable with CSS and scriptable; not for photographic detail.

Rule of thumb: photos → JPEG/WebP; logos/icons/UI with transparency → PNG/WebP/SVG; animations → GIF (legacy) or animated WebP/SVG where supported.
